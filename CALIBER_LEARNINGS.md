# Caliber Learnings

Accumulated patterns and anti-patterns from development sessions.
Auto-managed by [caliber](https://github.com/caliber-ai-org/ai-setup) — do not edit manually.

- **[gotcha]** `caliber sync` is NOT a valid subcommand — use `caliber refresh` instead. CLAUDE.md and `.claude/rules/agent-config.md` both document `caliber sync` but the binary does not have this command (confirmed via `caliber --help`). Running `caliber sync` fails silently with empty output.
- **[pattern]** `caliber init` takes ~10 minutes — always run as a background task: `caliber init --auto-approve --agent claude,codex`. Wait for the background task notification rather than polling the output file.
- **[gotcha]** `caliber refresh --quiet` produces no stdout on success — empty result is normal. Verify via exit code or background task notification, not visible output.
- **[env]** MCP packages `@modelcontextprotocol/server-filesystem` and `@modelcontextprotocol/server-github` are installed globally at `/opt/homebrew/lib`. The `npx -y` in `.mcp.json` uses cached global versions without a network fetch.
- **[convention]** `statusLine` config belongs in `~/.claude/settings.json` (global user-level), NOT project `.claude/settings.json`. Project settings manages hooks; user settings manages UI config.
- **[env]** `.caliber/` directory is gitignored — caliber state (including `lastRefreshSha` for freshness hooks) is machine-local, never shared via git.
- **[gotcha]** `caliber init` output file contains raw ANSI escape sequences (spinner animations `\u001b[0J`, `\u001b[8A`, etc.) — grepping the output file for completion markers is unreliable. Use `pgrep -f caliber` to check if still running, or wait for background task notification.
- **[gotcha]** Files generated by `caliber init` — including `.claude/rules/agent-config.md` and `AGENTS.md` — contain stale `caliber sync` references. After every `caliber init`, check these files and replace `caliber sync` with `caliber refresh`.
- **[gotcha]** `caliber score --json` produces multi-line JSON — piping through `head -1` returns only `{`. Read the full output or omit `head -1`.
