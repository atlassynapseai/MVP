- **[gotcha]** `tsconfig.base.json` enables `noUncheckedIndexedAccess: true` ‚Äî array index access (`arr[i]`) returns `T | undefined`, not `T`. Always use `?? fallback` for positional array indexing (e.g., `digits[i] ?? "0"`). This silently breaks code written for standard TS.
- **[convention]** Import `Prisma` namespace types from `@atlas/db`, not directly from `@prisma/client` ‚Äî `packages/db/src/index.ts` re-exports everything via `export * from "@prisma/client"`. Use `import { prisma, Prisma } from "@atlas/db"` in `apps/web/` to avoid duplicate resolution in the turbo monorepo.
- **[convention]** CI must run 'pnpm --filter @atlas/db generate' (with dummy DATABASE_URL + DIRECT_URL env vars) BEFORE typecheck step ‚Äî Prisma types don't exist until generate runs, causing TS2694 errors on every schema change
- **[env]** Clerk env vars in `apps/web/.env.local` must NOT have surrounding quotes on values. `NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY="pk_test_..."` with literal quote characters causes `InvalidCharacterError: Invalid character` in Clerk's `atob`-based key parsing.
- **[convention]** pnpm workspaces: packages without eslint installed (`@atlas/shared`, `@atlas/edge`) must use `"lint": "tsc --noEmit"` not `"lint": "eslint src"` ‚Äî eslint is not installed in those packages.

- **[pattern:project]** Vercel Cron is the queue mechanism ‚Äî no pg-boss (requires persistent Node process, incompatible with Vercel serverless). Cron hits `/api/cron/evaluate` every 60s, polls DB for `status=received` traces. Batch size 5 per invocation.
- **[gotcha:project]** Vercel Cron routes must set `export const maxDuration = 60` ‚Äî default Vercel function timeout is 10s, too short for AI eval chains (~20s for 5 traces √ó 2 Claude calls each). Without this, cron jobs abort mid-evaluation.
- **[gotcha:project]** Clerk webhook event ordering: `organizationMembership.created` can arrive before `organization.created`. Always upsert Org from the nested `organization` field embedded in the membership event payload before creating the User row ‚Äî never assume org row already exists.
- **[convention:project]** Evaluator + alerting logic lives in `packages/evaluator/` (`@atlas/evaluator`). Add `@anthropic-ai/sdk` and `resend` to that package, NOT to `apps/web`. `apps/web` imports `@atlas/evaluator` as a workspace dependency (`workspace:*`).
- **[fix:project]** `pnpm --filter @atlas/db exec prisma migrate dev --name <name> --create-only` silently returns empty `{}` bash output (no error, no feedback) when `DATABASE_URL`/`DIRECT_URL` env vars are absent. Fix: either set vars in `packages/db/.env` OR manually create `packages/db/prisma/migrations/<timestamp>_<name>/migration.sql` + `packages/db/prisma/migrations/migration_lock.toml` (content: `provider = "postgresql"`).
- **[gotcha:project]** N8N AI Agent nodes have no native outbound execution webhook ‚Äî a reporter HTTP Request node must be manually added to each N8N workflow to send execution data to AtlasSynapse. N8N does not expose token counts natively; always set `tokenCount: null` for N8N traces.
- **[pattern:project]** On Agent upsert in `/api/ingest`, write `platform` field on `create` only (not on `update`) ‚Äî prevents overwriting a previously set platform value when the same agent sends subsequent traces.
- **[fix:project]** Vitest ESM mock pattern for `@anthropic-ai/sdk`: place `vi.mock("@anthropic-ai/sdk", ...)` before all static `import` statements ‚Äî Vitest hoists it. Do NOT use `await import()` inside synchronous helper functions after mock setup. Access the mock's `create` fn via `new MockedConstructor().messages.create`. Initial test with top-level `await import()` in sync helpers caused silent tsc failure (empty `{}` output from typecheck tool).
- **[gotcha:project]** Prisma optional nullable fields: use `null` not `undefined` in `create`/`update` data objects ‚Äî `field ?? undefined` fails TS because `undefined` is not assignable to `T | null`. Always write `field ?? null` for optional nullable Prisma columns.
- **[env:project]** Python SDK tests in `packages/sdk-python/`: run as `PYTHONPATH=src python3 -m pytest tests/` ‚Äî editable install (`pip install -e .`) fails on macOS system Python 3.9 (pip 21 doesn't support pyproject.toml-only builds without setuptools). The `PYTHONPATH=src` workaround is documented in `pyproject.toml`.
- **[fix:project]** Vitest ESM mock pattern for `@anthropic-ai/sdk`: place `vi.mock("@anthropic-ai/sdk", ...)` before all static `import` statements ‚Äî Vitest hoists it. Do NOT use `await import()` inside synchronous helper functions after mock setup. Access the mock's `create` fn via `new MockedConstructor().messages.create`.
- **[gotcha:project]** Shell `[id]` in paths (Next.js dynamic route dirs like `incidents/[id]/`) is treated as a zsh glob pattern ‚Äî causes `no matches found` in both `mkdir -p` and `git add`. Always quote these paths: `mkdir -p "path/[id]"` and `git add "apps/web/app/.../[id]/"`.
- **[fix:project]** Prisma `findUnique` with compound unique key `orgId_agentId` (where `agentId` is `String?`) fails TypeScript ‚Äî `null` is not assignable to `string` in the composite key object. Use `findFirst({ where: { orgId, agentId: null } })` instead of `findUnique({ where: { orgId_agentId: { orgId, agentId: null } } })`.
- **[gotcha:project]** Next.js 15 App Router: dynamic segment `params` prop type is `Promise<{ id: string }>` ‚Äî must `await params` before destructuring. Pattern: `interface Props { params: Promise<{ id: string }> }` then `const { id } = await params` inside the async page component.
- **[gotcha:project]** Clerk middleware blocks ALL routes by default, including non-Clerk-authenticated API routes. Any route using its own auth (HMAC, Svix, bearer token) must be added to `isPublicRoute` in `apps/web/middleware.ts` ‚Äî e.g., `/api/webhooks/(.*)` and `/api/cron/(.*)`. Omitting them causes 401 before the route handler even runs.
- **[gotcha:project]** Cron guard must be fail-closed: `if (!cronSecret) return 500` THEN `if (authHeader !== bearer) return 401`. The fail-open pattern `if (cronSecret) { check }` leaves the endpoint unprotected when `CRON_SECRET` env var is unset ‚Äî any caller gets through.
- **[pattern:project]** API routes accepting a user-supplied resource ID (e.g. `agentId` query param or body field) must verify org ownership before use: `prisma.agent.findFirst({ where: { id: agentId, orgId: org.id } })`. Skipping this creates IDOR ‚Äî authenticated users from org A can read/write org B's resources.
