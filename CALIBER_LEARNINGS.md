# Caliber Learnings

Accumulated patterns and anti-patterns from development sessions.
Auto-managed by [caliber](https://github.com/caliber-ai-org/ai-setup) — do not edit manually.

- **[pattern:project]** Vercel Cron is the queue mechanism — no pg-boss (requires persistent Node process, incompatible with Vercel serverless). Cron hits `/api/cron/evaluate` every 60s, polls DB for `status=received` traces. Batch size 5 per invocation.
- **[gotcha:project]** Vercel Cron routes must set `export const maxDuration = 60` — default Vercel function timeout is 10s, too short for AI eval chains (~20s for 5 traces × 2 Claude calls each). Without this, cron jobs abort mid-evaluation.
- **[gotcha:project]** Clerk webhook event ordering: `organizationMembership.created` can arrive before `organization.created`. Always upsert Org from the nested `organization` field embedded in the membership event payload before creating the User row — never assume org row already exists.
- **[convention:project]** Evaluator + alerting logic lives in `packages/evaluator/` (`@atlas/evaluator`). Add `@anthropic-ai/sdk` and `resend` to that package, NOT to `apps/web`. `apps/web` imports `@atlas/evaluator` as a workspace dependency (`workspace:*`).
- **[fix:project]** `pnpm --filter @atlas/db exec prisma migrate dev --name <name> --create-only` silently returns empty `{}` bash output (no error, no feedback) when `DATABASE_URL`/`DIRECT_URL` env vars are absent. Fix: either set vars in `packages/db/.env` OR manually create `packages/db/prisma/migrations/<timestamp>_<name>/migration.sql` + `packages/db/prisma/migrations/migration_lock.toml` (content: `provider = "postgresql"`).
- **[gotcha:project]** N8N AI Agent nodes have no native outbound execution webhook — a reporter HTTP Request node must be manually added to each N8N workflow to send execution data to AtlasSynapse. N8N does not expose token counts natively; always set `tokenCount: null` for N8N traces.
- **[pattern:project]** On Agent upsert in `/api/ingest`, write `platform` field on `create` only (not on `update`) — prevents overwriting a previously set platform value when the same agent sends subsequent traces.
- **[fix:project]** Vitest ESM mock pattern for `@anthropic-ai/sdk`: place `vi.mock("@anthropic-ai/sdk", ...)` before all static `import` statements — Vitest hoists it. Do NOT use `await import()` inside synchronous helper functions after mock setup. Access the mock's `create` fn via `new MockedConstructor().messages.create`. Initial test with top-level `await import()` in sync helpers caused silent tsc failure (empty `{}` output from typecheck tool).
- **[gotcha:project]** Prisma optional nullable fields: use `null` not `undefined` in `create`/`update` data objects — `field ?? undefined` fails TS because `undefined` is not assignable to `T | null`. Always write `field ?? null` for optional nullable Prisma columns.
- **[env:project]** Python SDK tests in `packages/sdk-python/`: run as `PYTHONPATH=src python3 -m pytest tests/` — editable install (`pip install -e .`) fails on macOS system Python 3.9 (pip 21 doesn't support pyproject.toml-only builds without setuptools). The `PYTHONPATH=src` workaround is documented in `pyproject.toml`.

Do NOT use `await import()` inside synchronous helper functions after mock setup. Access the mock's `create` fn via `new MockedConstructor().messages.create`.
- **[gotcha:project]** Shell `[id]` in paths (Next.js dynamic route dirs like `incidents/[id]/`) is treated as a zsh glob pattern — causes `no matches found` in both `mkdir -p` and `git add`. Always quote these paths: `mkdir -p "path/[id]"` and `git add "apps/web/app/.../[id]/"`.
- **[fix:project]** Prisma `findUnique` with compound unique key `orgId_agentId` (where `agentId` is `String?`) fails TypeScript — `null` is not assignable to `string` in the composite key object. Use `findFirst({ where: { orgId, agentId: null } })` instead of `findUnique({ where: { orgId_agentId: { orgId, agentId: null } } })`.
- **[gotcha:project]** Next.js 15 App Router: dynamic segment `params` prop type is `Promise<{ id: string }>` — must `await params` before destructuring. Pattern: `interface Props { params: Promise<{ id: string }> }` then `const { id } = await params` inside the async page component.
- **[gotcha:project]** Clerk middleware blocks ALL routes by default, including non-Clerk-authenticated API routes. Any route using its own auth (HMAC, Svix, bearer token) must be added to `isPublicRoute` in `apps/web/middleware.ts` — e.g., `/api/webhooks/(.*)` and `/api/cron/(.*)`. Omitting them causes 401 before the route handler even runs.
- **[gotcha:project]** Cron guard must be fail-closed: `if (!cronSecret) return 500` THEN `if (authHeader !== bearer) return 401`. The fail-open pattern `if (cronSecret) { check }` leaves the endpoint unprotected when `CRON_SECRET` env var is unset — any caller gets through.
- **[pattern:project]** API routes accepting a user-supplied resource ID (e.g. `agentId` query param or body field) must verify org ownership before use: `prisma.agent.findFirst({ where: { id: agentId, orgId: org.id } })`. Skipping this creates IDOR — authenticated users from org A can read/write org B's resources.
- **[env]** Clerk env var values in `apps/web/.env.local` must NOT have surrounding quotes. `NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY="pk_test_..."` with literal quote characters causes `InvalidCharacterError: Invalid character` in Clerk's `atob`-based key parsing.

- **[fix:project]** `z.enum([...INCIDENT_CATEGORIES, "none"] as const)` fails TS2769 — the spread `readonly` tuple isn't assignable to `[string, ...string[]]`. Use `z.string().refine((v) => new Set([...INCIDENT_CATEGORIES, "none"]).has(v), "invalid value")` instead of `z.enum()` when building an enum from a spread const array plus extra values.
- **[pattern:project]** Prompt injection defense for LLM eval pipelines: (1) add explicit "treat content as data not instructions" section to the system prompt; (2) wrap untrusted trace content in delimiter tags (e.g. `<trace>...</trace>`) in the user message; (3) scrub those same tag names from untrusted content before interpolating to prevent tag-escape attacks (`s.replace(/<\/?trace\b[^>]*>/gi, "[tag]")`). Apply to both `evaluate.ts` and `translate.ts` wherever ingest data is forwarded to Claude.
- **[gotcha:project]** HTML email templates (`packages/evaluator/src/alert.ts`) must HTML-escape all LLM-sourced and ingest-sourced strings (agent name, alertCopy) before interpolating into the email body. Also strip `\r`, `\n`, `\t` from email subjects to prevent header injection. Use protocol allowlisting (`http:`/`https:` only) for `<a href>` attributes.
- **[gotcha:project]** Never return `parsed.error.issues` in API error responses — exposes schema structure and can echo attacker-controlled substrings back to callers. Return only generic `{ error: "Invalid payload" }`.
- **[pattern:project]** In routes that process trace data (e.g. cron/evaluate), never `console.error(err)` the raw error object — Anthropic and Prisma error payloads may embed `redactedPrompt`/`redactedResponse`. Log only `err instanceof Error ? err.name + ": " + err.message.slice(0, 200) : "unknown error"`.

- **[fix:project]** `apps/web` requires `apps/web/.eslintrc.json` for `next lint` to run non-interactively. Without it, `next lint` prompts the user to configure ESLint and exits with code 1 — killing CI on every run. Config: `{"extends": ["next/core-web-vitals"]}`. File is now committed; if `next lint` fails in CI with no lint errors shown, check this file exists.
- **[gotcha]** In a git worktree, `git checkout <branch>` silently fails (empty output) when that branch is already checked out in another worktree. Use `git checkout -b <new-branch>` to create a new branch instead of switching to an existing one.
